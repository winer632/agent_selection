AgentScope 技术架构介绍（2026）

本文基于 AgentScope 最新源码，按“运行时基础 → Agent 核心执行 → 协议/生态扩展 → 工程闭环”拆解核心架构， 聚焦 ReAct 主链路、Tool/MCP/A2A 接口、Memory/RAG、Realtime/TTS、多 Agent 编排与评测调优体系。

代码基线：agentscope@28cfb99 更新时间：2026-03-04 范围：Core SDK + Integrations + Evaluate + Tuner
一句话概括： AgentScope 是“以 ReAct 运行时为中心、以协议适配和工程闭环为两翼”的通用 Agent SDK，不是单体网关，而是可嵌入任意业务应用的执行框架。

多 Agent 支持现状

结论（截至 2026-03-04）：已支持且为当前能力，不是 roadmap。

  • 框架内置 MsgHub + Pipeline，可做多 Agent 编排与消息路由。
  • 官方样例已覆盖 multiagent_conversation、multiagent_debate、multiagent_concurrent、multiagent_realtime。
  • 支持 A2A 协议，可做跨 Agent 协作与对接。

架构总览与分层

Layer A

运行时基础层

全局 run context、StateModule、Session 存储与可序列化状态边界。

Layer B

Agent 执行核心层

AgentBase/ReAct、Model/Formatter、Toolkit、Memory、PlanNotebook 形成主执行回路。

Layer C

扩展协作层

RAG、MCP、A2A、Pipeline、Realtime/TTS 负责跨系统互联与多模态交互。

Layer D

工程闭环层

Tracing、Evaluate、Tuner 构成可观测、可评估、可优化的持续迭代闭环。

ReAct 执行内核深度解析

AgentScope 的核心增量不在“固定工作流 DSL”，而在可扩展 ReAct 主循环： 把工具调用、计划、记忆、结构化输出与多模态输出融合到同一个 reply 状态机中。

ReAct-01

输入阶段：先写记忆再执行

  • reply 开始就把输入消息写入 working memory。
  • 随后触发长期记忆检索与知识检索增强。

src/agentscope/agent/_react_agent.py

ReAct-02

结构化输出模式是“工具化约束”

  • 当传入 structured_model 时，动态注册 generate_response 工具。
  • 通过 tool_choice="required" 强制模型走结构化产出路径。

src/agentscope/agent/_react_agent.py

ReAct-03

循环核心：reasoning → acting

  • 每轮先 reasoning 产出文本/工具调用，再 acting 执行工具。
  • 支持顺序执行与并行执行（parallel_tool_calls）。

src/agentscope/agent/_react_agent.py

ReAct-04

Reasoning 内部整合 Plan 与 TTS

  • PlanNotebook 可先注入系统提示，指导下一步动作。
  • 模型输出支持流式 print，并可对接 TTS 流式合成。

src/agentscope/plan/_plan_notebook.pysrc/agentscope/agent/_react_agent.pysrc/agentscope/tts/_tts_base.py

ReAct-05

Acting 阶段统一为 ToolResponse 流

  • Toolkit 调用返回统一流式 ToolResponse。
  • 每个 chunk 都可即时打印并回写 memory。

src/agentscope/tool/_toolkit.pysrc/agentscope/agent/_react_agent.py

ReAct-06

记忆压缩与异常回退

  • 当迭代超过阈值或上下文过大，可触发压缩策略。
  • 达到 max_iters 仍无最终答复时，进入 summarizing fallback。

src/agentscope/agent/_react_agent.py

静态长期记忆回写

  • 在 static_control/both 模式下，reply 结束后将会话回写 long-term memory。
  • 形成“执行后沉淀”的记忆闭环。

关键运行链路

Flow A

单轮 ReAct 主链路

  • 输入入库（working memory）→ 长期记忆与知识检索。
  • Formatter 组装 prompt → Model 产出。
  • Toolkit 执行工具并回灌 memory，直到生成最终回复。
Flow B

MCP 工具桥接链路

  • MCP Client 获取 remote tools。
  • Toolkit 注册为本地可调用函数并暴露 schema 给模型。
  • 模型通过 tool_use 触发调用并接收 tool_result。
Flow C

工程闭环链路

  • Tracing 装饰器在 agent/model/tool/formatter 上打点。
  • Evaluate 对 benchmark/task/metric 聚合结果。
  • Tuner 将 workflow/judge/dataset 转为训练配置进行优化。

工程特征

  • 以执行循环为中心：ReAct 是统一主路径，功能通过参数与组件扩展，不靠分裂 runtime。
  • 协议适配优先：Model/Formatter/MCP/A2A 都走抽象层，降低厂商耦合。
  • 状态可迁移：StateModule + Session 让 agent 运行态具备持久化与恢复能力。
  • 多模态和实时并行推进：Realtime、TTS、Pipeline 与传统文本 Agent 共存。
  • 可观测可评测可调优：Tracing + Evaluate + Tuner 构成长期演进基础设施。
